Useful GIT Commands:
    git clone <repository url> - downloads the repository onto your computer locally
    Make sure to CD to the repository on your local computer before running any of the later commands. 

    Committing: To sync changed from local computer to the shared repository, use following commands to 
    commit your changes to the shared repository: 
    First, make sure you save the files you have edited
    git add <file> - stages a specific file for committing (if you only edited one file)
    git add ./ - stages all files in the working directory for committing (if you edited or created multiple files)
    git commit -m "<commit message>" - compiles all the files for comitting. Be sure to add a message at the end describing the edits or commits you have done
    git push origin main - commits all files in the staging phase to the shared repository


    Updating local code to reflect shared repository (MAKE SURE TO DO THIS EVERYTIME WHEN STARTING CODE AS OTHERS MAY HAVE DONE EDITS TO THE REPOSITORY):git
    git fetch
    git pull
